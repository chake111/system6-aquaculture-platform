## Project

Vue 3 SPA for the System 6 aquaculture monitoring platform. Provides role-based dashboards for water quality monitoring, alert workflow management, density analysis, benefit reporting, and operations management.

## Commands

```bash
npm install                  # install dependencies
npm run dev                  # vite dev server (proxies /api to localhost:8000)
npm run build                # vue-tsc --build + vite build
npm run preview              # vite preview of production build
npm run lint                 # oxlint then eslint
npm run lint:fix             # auto-fix lint issues
npm run format               # prettier write
npm run format:check         # prettier check (used in CI)
npm run test:unit            # vitest (watch mode)
npm run test:unit -- --run   # vitest single run (for CI)
npm run type-check           # vue-tsc --build
```

## Architecture

### Entry and routing

- **main.ts** ‚Äî Creates Vue app, installs Pinia and Vue Router.
- **router/index.ts** ‚Äî Routes under two layouts: standalone `/login` and `AppShellView` wrapper for `/dashboard`, `/monitoring`, `/alerts`, `/reports`, `/operations`, `/density`. Role guard redirects unauthorized roles to `/dashboard`. Offline guard allows only `/dashboard` and `/monitoring` with a valid cached offline grant.

### State management

- **stores/platform.ts** ‚Äî Single Pinia store (`usePlatformStore`) owns all app state. Composes auth, observations, alert workflow, reports, operations, density, and agent advice. All API calls go through `createRequest()`. Actions manage multi-step workflows (e.g., alert resolution is a 3-step sequence: feedback ‚Üí acknowledge ‚Üí resolve).

### Utilities

- **utils/api.ts** ‚Äî `createRequest(getToken, onUnauthorized)` returns a typed `fetch` wrapper. Attaches `Authorization: Bearer` header, handles 401 redirect, throws on non-OK responses.
- **utils/loading.ts** ‚Äî `createLoading()` provides per-action loading flags via `withLoading(actionName, fn)`.
- **utils/storage.ts** ‚Äî `STORAGE_KEYS` constants and `storedValue<T>()` for typed sessionStorage reads.

### Types

- **types/domain.ts** ‚Äî Core domain types: `UserRole`, `Observation`, `OxygenAlert`, `AlertStatus`, `BenefitReport`, `DensityResult`, `AgentAdvice`, `OfflineGrant`, etc.
- **types/auto-imports.d.ts** / **types/components.d.ts** ‚Äî Auto-generated by `unplugin-auto-import` and `unplugin-vue-components`. Do not edit manually; regenerated on dev server start.

### Views and components

- **views/** ‚Äî Page-level components. `AppShellView` is the authenticated layout with sidebar navigation. Each view maps to a route.
- **components/** ‚Äî `AgentPanel` (LLM advice display), `TrendChart` (SVG-based data visualization), `VoiceLogin` (speech recognition login).

### Element Plus

Auto-imported via `unplugin-vue-components` + `unplugin-auto-import` with `ElementPlusResolver`. No manual imports needed for Element Plus components or composables.

### API proxy

Vite dev server proxies `/api` requests to `http://localhost:8000` (backend). Configured in `vite.config.ts`.

### Testing

- Tests live in `src/__tests__/`.
- Vitest environment is `jsdom`. Element Plus is inlined in server deps to avoid transform issues.
- Tests mock `fetch` globally via `vi.stubGlobal('fetch', ...)`.
- Mount helpers create Pinia + router, push to `/login`, and flush promises.

## Code Style

- 2-space indent, LF line endings, UTF-8.
- Linting: Oxlint runs first (fast), then ESLint catches what Oxlint misses. ESLint config chains Vue essential + TypeScript + Vitest + Oxlint integration + Prettier compat.
- Prettier for formatting; ESLint does not format.
