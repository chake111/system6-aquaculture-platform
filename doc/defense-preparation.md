# 答辩准备材料

## 系统概述

智慧水产养殖监测与调控平台，基于 Vue 3 + FastAPI + SQLite 实现，覆盖水质监测、溶氧调控、预警处置、密度分析、效益报表和运维管理六大模块。支持语音登录，适配鱼塘潮湿环境操作。

---

## 答辩问题与参考回答

### 问题 1：为什么选择这个技术栈？

**参考回答：**

前端选择 Vue 3 + TypeScript + Vite：
- Vue 3 组合式 API 适合复杂状态管理
- TypeScript 提供类型安全，减少运行时错误
- Vite 开发体验好，热更新快

后端选择 Python + FastAPI：
- Python 在数据处理和 AI 领域生态丰富
- FastAPI 自动生成 OpenAPI 文档，开发效率高
- SQLAlchemy ORM 简化数据库操作

数据库选择 SQLite：
- 演示环境无需额外安装数据库服务
- 进程内存储，重启自动初始化演示数据
- 生产环境可无缝切换到 PostgreSQL/MySQL

### 问题 2：系统的核心业务流程是什么？

**参考回答：**

核心是"溶氧预警处置闭环"：

1. **监测**：实时采集水质数据（溶氧、pH）
2. **预警**：溶氧低于阈值（5.0 mg/L）自动触发告警
3. **建议**：AI 或规则引擎生成增氧建议
4. **复核**：技术员审核建议合理性
5. **确认**：养殖户确认执行
6. **执行**：模拟/真实启动增氧机
7. **反馈**：记录执行后溶氧变化
8. **关闭**：技术员确认处置完成

整个流程有完整的审计追踪，每一步都记录操作人、时间和证据。

### 问题 3：如何保证数据的真实性和可追溯性？

**参考回答：**

系统通过三层机制保证：

1. **来源标识**：每条数据都有 `source_mode` 字段
   - `crawled`：通过爬虫脚本尝试从公开 API 采集，降级时基于真实参数范围生成
   - `simulation`：演示模拟数据
   - `auto`：自动采集（生产环境）

2. **数据来源**：广西惠州鲈鱼养殖基地（GX-HZ-001）
   - 溶氧范围：5.1-7.8 mg/L（含日变化规律，符合广西鲈鱼养殖塘口特征）
   - pH 范围：7.0-7.7（弱碱性，符合广西淡水养殖水质标准）
   - 数据周期：24小时逐小时采集
   - 采集方式：`scripts/fetch_guangxi_data.py` 脚本尝试从广西公共数据开放平台、生态环境部数据中心等公开 API 采集；这些政府平台均为动态 JavaScript 应用，未暴露标准 REST API，因此基于广西生态环境厅公开的水质参数范围生成数据

2. **审计日志**：所有操作记录在 `audit_logs` 表
   - 操作人 ID
   - 操作类型
   - 操作对象
   - 时间戳
   - 请求追踪 ID

3. **脱敏导出**：报表导出时自动脱敏
   - 采用 `mask-identifiers-v1` 策略
   - 导出记录有唯一 ID 和到期时间
   - 支持归档审批流程

## 技术亮点

1. **角色权限隔离**：三种角色（养殖户、技术员、管理员）权限严格分离
2. **状态机管理**：建议和告警有完整的状态流转（generated → reviewed → confirmed → executed → evaluated → resolved → closed）
3. **模拟/真实分离**：所有模拟数据明确标识，不与真实数据混淆
4. **离线支持**：支持离线授权和缓存，弱网环境可降级使用
5. **AI 集成**：支持 DeepSeek 大模型生成养殖建议，无 API 时自动降级到规则引擎
