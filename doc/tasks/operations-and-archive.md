# 运维与归档模块最小任务

> 模块范围：`FR-10`、`FR-12` 及 `FR-11` 的运维审计查询；支撑 `AT-10`，归档为 P2。
> 前置依赖：健康接口已存在；运维汇总在相应业务模块提供同步、通知、导出和审计数据后逐步接入。

**完成定义：** 管理员可查看服务/边缘/通知/同步健康状态和审计事件；可审计地维护阈值配置；归档/销毁仅依据审批引用创建证据记录；敏感配置与数据不在前端或日志中泄露。

## Checklist

### P01 运维健康与配置

- [ ] 实现管理员专用的 `GET /api/v1/operations/health`，聚合 API、数据库、边缘节点、通知任务、对象存储和同步积压的可诊断状态。
- [ ] 实现 `PUT /api/v1/threshold-rules/{id}` 的版本化启停和变更审计；规则值由业务确认后录入，演示配置明确标注用途。
- [ ] 从环境配置读取数据库、令牌、对象存储和外部渠道凭据，确保 API 响应、异常和审计元数据不输出秘密值。
- [ ] 编写测试验证仅管理员可访问运维/规则变更、健康状态降级可见、配置变更产生审计且秘密字段被遮蔽。

### P02 审计查询与安全证据

- [ ] 实现 `GET /api/v1/audit-logs` 的动作、资源、时间筛选和分页，限制为管理员权限并遮蔽敏感元数据。
- [ ] 确认登录、授权、建议执行、预警处置、外部通知、同步拒绝、导出与配置变更都通过统一审计写入接口落盘。
- [ ] 编写安全测试覆盖跨基地越权、普通用户查询审计、模拟/真实标识混淆、重放或重复幂等操作。
- [ ] 准备数据库备份与恢复演练命令/记录模板，使用演示数据验证可恢复性并确保备份文件遵循敏感数据访问限制。
- [ ] 为性能与可靠性验收准备记录位置：登录压测、查询耗时、异常到送达时间、断网补传和组件健康结果。

### P03 前端管理面板

- [ ] 创建仅管理员可访问的运维页面，展示服务/节点/消息/同步状态、失败原因、最后更新时间和模拟环境标识。
- [ ] 实现阈值规则版本列表与启停操作、审计查询筛选和敏感内容遮蔽；变更操作使用二次确认。
- [ ] 将基地/鱼塘/设备维护入口与数据模块能力组合到管理页面，避免向普通角色显示管理操作。
- [ ] 编写组件测试验证管理员路由、故障状态、规则确认、审计筛选与敏感遮蔽展示。

### P04 归档/销毁与交付证据

- [ ] 创建 `archive_records` 模型和受管理员保护的归档/销毁记录用例，要求审批引用、数据范围、动作类型、执行人、时间和证据引用齐全。
- [ ] 为导出文件归档和批准后的销毁生成审计事件；演示阶段只验证流程和证据，不虚构监管接入或真实销毁结论。
- [ ] 编写部署、五分钟演示和验收结果文档，引用可运行命令、测试数据模式和实际取得的测试证据。
- [ ] 编写 API 测试验证缺少审批引用时拒绝归档/销毁、重复请求幂等以及归档记录不可由普通业务接口删除。

### P05 模块验收

- [ ] 运行 `npm run backend:test -- tests/unit/test_operations_service.py tests/api/test_operations.py`。
- [ ] 运行 `npm run frontend:test -- src/__tests__/OperationsView.spec.ts`。
- [ ] 运行 `npm run check`，确认完整前后端质量检查通过。
- [ ] 完成一次管理员演练：查看故障/积压 -> 查询审计 -> 变更演示阈值 -> 发起脱敏导出归档记录，并将实际结果写入验收文档。
